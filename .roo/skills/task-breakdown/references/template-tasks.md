# 任务模板参考

## 文件结构

```markdown
---
template: Tasks
version: 1.1.0
lastUpdated: YYYY-MM-DD
charter: ../constitution.md
---

# 任务：[功能名称]

**输入**: 设计文档  
**先决条件**: plan.md、spec.md

---

## 任务标识

`[T-ID] [优先级] [并行标记] [用户故事] 文件路径 描述`

### 优先级

| 符号 | 含义 | 说明 |
|------|------|------|
| P0 | 阻塞 | 必须优先完成 |
| P1 | 高 | 核心功能 |
| P2 | 中 | 重要功能 |
| P3 | 低 | 增强功能 |

### 并行标记

| 符号 | 含义 | 说明 |
|------|------|------|
| [P] | 可并行 | 不同文件，无依赖关系 |

---

## 阶段1：设置

- [ ] T001 [P0] 创建项目结构

## 阶段2：基础

- [ ] T002 [P0] 设置数据库模式

## US1：[标题]（P1）

**目标**: [简要描述]

### 测试

- [ ] T003 [P1] [P] [US1] 编写契约测试

### 实施

- [ ] T004 [P1] [US1] 实现功能
```

## 任务标识说明

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| T-ID | 任务编号 | T001, T002 |
| 优先级 | P0/P1/P2/P3 | P0, P1 |
| 并行标记 | [P] 或留空 | [P] |
| 用户故事 | US1/US2/US3 | [US1] |
| 文件路径 | 目标文件位置 | src/models/user.py |
| 描述 | 任务简述 | 创建用户模型 |

## 阶段说明

| 阶段 | 目的 | 依赖 |
|------|------|------|
| 阶段1：设置 | 项目初始化 | 无 |
| 阶段2：基础 | 阻塞性先决条件 | 设置完成 |
| US1/US2 | 用户故事实施 | 基础完成 |
| 阶段N：完善 | 横切关注点 | 所有故事完成 |

## 任务数量参考

| 阶段 | 任务数量参考 |
|------|-------------|
| 阶段1：设置 | 2-5 个任务 |
| 阶段2：基础 | 4-8 个任务 |
| 每个用户故事 | 5-10 个任务 |
| 阶段N：完善 | 3-6 个任务 |

## 注意事项

- P0 任务必须最先完成
- [P] 任务可并行执行
- 测试任务先于实现任务
- 文档任务包含在任务列表中
