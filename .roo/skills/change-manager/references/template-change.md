# 变更模板参考

## 文件结构

```markdown
---
template: Change
version: 1.0.0
lastUpdated: YYYY-MM-DD
charter: ../constitution.md
---

# 变更提案：[变更名称]

**变更ID**: [kebab-case-标识符]  
**创建日期**: [YYYY-MM-DD]  
**状态**: 草稿/已批准/已实施/已归档  
**关联功能**: [spec.md 链接]

---

## 变更原因

[用 1-2 句话解释为什么需要这个变更]

## 变更内容

- **[功能模块]**: [简要描述变更]

**重大变更**: 如有破坏性变更，请用 **BREAKING** 标记

## Delta 操作

### 新增需求 (ADDED)

- **[新增需求描述]**

### 修改需求 (MODIFIED)

- **[原需求描述] → [新需求描述]**

### 移除需求 (REMOVED)

- **[移除的需求描述]**

### 重命名需求 (RENAMED)

- **[原名称] → [新名称]**

## 影响范围

**受影响的规格**:
- [关联的 spec.md 文件]

**受影响的代码**:
- [关键文件路径]

---

## 实施计划

### 阶段 1：准备

- [ ] C001 审查现有规格文档

### 阶段 2：实施

- [ ] C002 更新规格 delta

### 阶段 3：验证

- [ ] C003 运行验证命令
```

## Delta 操作类型

| 类型 | 说明 | 示例 |
|------|------|------|
| ADDED | 新增需求 | 新增支付功能 |
| MODIFIED | 修改需求 | 修改 API 响应格式 |
| REMOVED | 移除需求 | 移除旧版接口 |
| RENAMED | 重命名 | 重命名字段或接口 |

## 变更状态

| 状态 | 说明 |
|------|------|
| 草稿 | 变更提案创建中 |
| 已批准 | 变更已获得审批 |
| 已实施 | 变更已实现 |
| 已归档 | 变更已合并并归档 |

## 影响范围分析

| 维度 | 分析内容 |
|------|----------|
| 受影响的规格 | 关联的功能规格文档 |
| 受影响的代码 | 关键文件路径、API 端点 |
| 受影响的依赖 | 上下游系统、第三方服务 |
| 兼容性影响 | 是否破坏向后兼容 |

## 注意事项

- 重大变更必须标记 **BREAKING**
- 必须获得人工批准后才能实施
- 需要更新所有受影响的文档
- 实施后需要验证和测试
