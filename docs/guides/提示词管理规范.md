# 提示词管理规范

> 
> **单一事实来源 (Source of Truth)**: [`speckit/specs/prompt-management-standards.md`](../../speckit/specs/prompt-management-standards.md)

## 1. 核心原则

### 禁止硬编码 (No Hard-coding)
- **严禁**在业务逻辑代码（Service/Controller）中直接拼接 Prompt 字符串。
- 所有 Prompt 必须通过配置化方式管理。

### 数据库优先 (DB First)
- 运行时系统**必须**优先从数据库加载 Prompt。
- 代码中的常量仅作为**初始化种子 (Seed)**。

## 2. 开发流程

1.  **定义**: 在业务模块下创建 `prompts.py` 定义 Prompt Seed。
2.  **注册**: 在 `src/shared/constants/prompts.py` 中引入并注册。
3.  **使用**: 在 Service 中注入 `PromptService` 获取 Prompt。

详细代码示例请参考 [技术规格文档](../../speckit/specs/prompt-management-standards.md)。
