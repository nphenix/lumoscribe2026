# 规格模板参考

## 文件结构

```markdown
---
template: Spec
version: 1.0.0
lastUpdated: YYYY-MM-DD
charter: ../constitution.md
---

# 功能规格说明：[功能名称]

**功能分支**: `feature/[issue-id]-[name]`  
**创建日期**: [YYYY-MM-DD]  
**状态**: 草稿  
**输入**: "[用户原始需求描述]"

---

## 用户场景与测试

### 用户故事 1 - [简短标题]（P1）

**验收场景**:
1. 假设 [初始状态]，当 [操作]，则 [预期结果]

## 需求

### 功能性需求

- **FR-001**: 系统必须 [具体能力]

### 关键实体

- **[实体1]**: [它代表什么；关键属性]
```

## 关键字段

| 字段 | 说明 | 示例 |
|------|------|------|
| template | 模板名称 | Spec |
| version | 模板版本 | 1.0.0 |
| lastUpdated | 最后更新日期 | 2026-01-12 |
| charter | 章程文件路径 | ../constitution.md |

## 用户故事优先级

| 优先级 | 说明 | 用途 |
|--------|------|------|
| P1 | 核心功能 | MVP 必须包含 |
| P2 | 重要功能 | 迭代 2 实现 |
| P3 | 增强功能 | 后续迭代 |

## 验收场景格式

使用"假设-当-则"结构：

```
假设 [初始状态]，当 [操作]，则 [预期结果]
```

## 注意事项

- 用户故事必须可独立测试
- 每个故事至少包含 2 个验收场景
- 边界和异常情况必须覆盖
